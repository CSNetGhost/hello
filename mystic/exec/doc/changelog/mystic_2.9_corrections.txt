* Corrected an issue where a crash could occur on review of the AER timestamp at 5 seconds. (:issue:`AFSIM-2329`)
* Corrected an issue where weapon-fire events would not affect :doc:`trace-lines<../wkf_plugin/rv_platform_history>`. (:issue:`AFSIM-1005`)
* Corrected an issue preventing exclusion zones within :doc:`zone sets<../zone_set>` from being displayed in Warlock and Mystic. (:issue:`AFSIM-1115`)
* Clicking "Clear Recents" button on the :doc:`Start-up Dialog <../mystic_start_dialog>` will clear the :doc:`File\>Open Recent <../mystic_file_menu>` menu. (:issue:`AFSIM-1116`)
* Corrected an issue where :command:`zones<zone>` with :command:`reference platforms<zone.reference_platform>` would not follow their platform during simulation in Warlock and Mystic. (:issue:`AFSIM-1333`)
* Fixed an issue where a :command:`zone` attached to a :command:`platform` with :command:`platform.use_zone` had default :command:`zone.line_color` and :command:`zone.fill_color` values instead of inheriting them. (:issue:`AFSIM-1350`)
* Corrected a crash that occurred when trying to load a non-aer file via the :doc:`Start-up Dialog <../mystic_start_dialog>`. (:issue:`AFSIM-1407`)
* Corrected the sign of range rate that is reflected in the :doc:`RelativeGeometry <../wkf_plugin/rv_relative_geometry>` plugin. (:issue:`AFSIM-1754`)
* Corrected an issue that prevented some data from plotting or displaying as a dragged overlay. (:issue:`AFSIM-2246`)
* Updated the event-pipe to represent time with a double-precision value. (:issue:`AFSIM-2295`)
* Comment output now clears itself at the start of each debug run. (:issue:`AFSIM-2120`)
