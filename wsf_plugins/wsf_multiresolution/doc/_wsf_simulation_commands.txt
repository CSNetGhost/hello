Multiresolution
^^^^^^^^^^^^^^^

Each multiresolution model defines a container for holding one or more models on a platform. 
The choice of which model to use is deferred to simulation time, and depends on the chosen fidelity value.

* :command:`multiresolution_comm` - Multiresolution container for comm models.
* :command:`multiresolution_fuel` - Multiresolution container for fuel models.
* :command:`multiresolution_mover` - Multiresolution container for movers.
* :command:`multiresolution_processor` - Multiresolution container for processors.
* :command:`multiresolution_sensor` - Multiresolution container for sensors.

* :command:`multiresolution_acoustic_signature` - Multiresolution container for acoustic signatures.
* :command:`multiresolution_infrared_signature` - Multiresolution container for infrared signatures.
* :command:`multiresolution_optical_signature` - Multiresolution container for optical signatures.
* :command:`multiresolution_radar_signature` - Multiresolution container for radar signatures.

* :command:`multiresolution_multirun_table` - Defines a multirun table of fidelity values. These fidelity values may be specified for any platform/model combination in the scenario. The multirun functionality works in conjunction with the :command:`final_run_number` command.
