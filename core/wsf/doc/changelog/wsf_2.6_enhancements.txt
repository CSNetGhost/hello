* **(2.6.2)** Added script method :method:`WsfGeoPoint.LocationECI`. *(Issue #2425)*
* **(2.6.2)** Added new script method :method:`Array\<T\>.Insert`, which can be used to insert one or more new elements into a script :class:`Array<Array\<T\>>` at a specified position. This method has replaced the undocumented method ``__Insert``. *(AFSIM-103)*
* Added support for English units of mass and length (``slug``, ``lb``, ``ft``, ``in``) when specifying angular inertia values.
* Added new script method :method:`WsfEM_XmtrRcvr.ArticulatedPart`. *(ELSZ #263)*
* Added new script method :method:`WsfPlatform.AddCategory` to add to a platform's :command:`platform.category` list. *(ELSZ #260)*
* Added the ability to select the :command:`radar_signature.interpolation_type` (linear or logarithmic) for signatures that use an :ref:`Azimuth-Elevation_Table_Definition`. *(Issue #1702)*
* Added new class :class:`Color` to represent colors in script, as well as a version of :method:`WsfDraw.SetColor` that accepts a Color object. The return type of :method:`WsfSimulation.ClassificationColor` has been changed from :class:`Vec3` to :class:`Color`. *(Issue #1803)*
* Added new script class :class:`Path` to represent file system paths. :method:`FileIO.Open` now accepts a Path object. *(Issue #1424)*
* The :command:`classification` and :command:`classification_levels` commands now support quoted strings for the level name, so the classification string can have spaces in it. The classification string created now uses "//" between trigraphs and caveats. *(Issue #1896)*
* Added new script block :command:`advanced_behavior_tree` that pairs with the new Behavior Tree Tool. *(Issue #1857)*
* Added the :command:`central_body<global_environment.central_body>` command block to the :command:`global_environment` inputs to allow changing the representation of the central body ellipsoid. *(Issue #1863)*
* Added the ability to publish :command:`visual_part` articulations through the :command:`event_pipe`. *(Issue #1913)*
* Added new command :command:`simulation_name` to facilitate simulation identification in Warlock and Results Vis. Name can be accessed from :method:`WsfSimulation.Name`. *(Issue #1931)*
* Added the scenario :command:`classification` string to the :command:`event_output` and :command:`csv_event_output` logs. *(Issue #1926)*
* Added new script method :method:`WsfEventPipe.AddBookmark` to add bookmarks to the scenario. *(Issue #2000)*
* Added new script classes :class:`WsfEM_XmtrRcvr`, :class:`WsfEM_Rcvr`, :class:`WsfEM_Antenna`, :class:`WsfEM_Attenuation`, :class:`WsfEM_Propagation`, and :class:`WsfAntennaPattern`, and added additional methods to :class:`WsfEM_Xmtr`. Added the ability to retrieve the :class:`WsfEM_Rcvr` from :class:`WsfSensor`, :class:`WsfComm`, and :class:`WsfEM_Interaction`. *(Issue #1821)*
* Added the ability to filter and format :command:`console_output`. Added variants of the :method:`writeln<__BUILTIN__.writeln>` script method to create different types of output. *(Issue #2146)*
* Added new sensor scheduler commands: :command:`scheduler_type.physical_scan.initial_heading` specifies the initial heading of the sensor, which would otherwise be random, for a :command:`scheduler_type.physical_scan` scheduler, and :command:`scheduler_type.sector_scan.frame_based_scheduling` configures the behavior of detection chances for a :command:`scheduler_type.sector_scan` scheduler. 
* Added :command:`medium` objects and associated capabilities to the communications framework. *(Issue #1309)*
