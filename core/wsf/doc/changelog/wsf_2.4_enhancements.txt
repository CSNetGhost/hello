* The :command:`event_pipe` will now publish start and stop events for detections and tracks from external DIS entities. *(Issue #796)*
* Added electrical responsivity as a new unit :ref:`(argument <argument_types>` type).  Responsivity has units of current / power. *(Issue #835)*
* Enumerated all event types for :command:`event_output<event_output.disable>`, :command:`csv_event_output<csv_event_output.disable>`, and :command:`observer<observer.disable>` in the grammar to facilitate auto-completion within Wizard. *(Issue #962)*
* Added Height-Above-Ellipsoid (HAE) functionality through script methods :method:`WsfGeoPoint.GeoidHeight`, :method:`WsfGeoPoint.HeightAboveEllipsoid` and :method:`WsfGeoPoint.SetAltitudeHAE`. *(Issue #1010)*
* Added a scriptable interface :class:`WsfEventPipe` that can publish data to the :command:`event_pipe` for visualization. *(Issue #1012)*
* Added in the capability to load the new NGA EGM Database via the :command:`gravity_model` input block. Legacy database support is removed in favor of the new formats. *(Issue #1029)*
* Added :ref:`docs/event/simulation_events:SIMULATION_STARTING` and :ref:`docs/event/simulation_events:SIMULATION_COMPLETE` events to :command:`event_output` and :command:`csv_event_output`. *(Issue #424)*
* Added sensor name to the SensorTrackCreated event-pipe message. *(Issue #1123)*
* Added :command:`route.aux_data` input commands to :command:`route` and supporting script methods to :class:`WsfRoute`. *(Issue #1218)*
* Added :method:`WsfPlatform.SetSide` to allow the user to specify the side of a platform prior to adding it to the simulation. *(Issue #1253)*
* Updated :class:`WsfRandom` and :class:`Math` to include a more comprehensive list of distributions for random number generation. *(ELSZ #18)*
* Updated :class:`Calendar` to handle leap seconds when setting the time. *(ELSZ #134)*
* Expanded :ref:`external link capabilities<External_Link_Commands>` to provide warnings of unresolved recipients during runtime when :command:`_.External_Link_Commands.debug_external_links` is enabled. *(Issue #1293)*
